对于相关性矩阵乘法与现实场景理解的重要更新！！！
大模型是二维矩阵乘以二维矩阵形成的矩阵乘法，也就是权重是三维。为什么是三维？因为它要考虑每个词先后的顺序造成最终结果的影响！第三个维度代表时间！
第一个维度，一维向量表示特征，有多少个特征，或者元素。第二维表示每个特征对各自造成的影响，二维就是计算元素之间的关联了！那么第三位呢？第三位代表时间，或者顺序，时间变换上的影响。也就是从M1*M2,代表了从M1开始到M2结束，产生时间跨越的累积影响（线性代数叫他变换）。如果从单个元素的视角看：每次变换都代表其他元素（包括自己在内全部）都对自己造成的单次全部影响X，第一次就是X1,第二次就是X2,n次就是Xn,X1*X2就是上次对这次每个其他元素造成的间接影响，再由其他元素对自己造成的直接影响产生的累积影响值。这就是非线性的计算累加影响。

  回到我们的agent每次任务产生的关键词序列，思路上：
由于每次任务之间不存在时间上的关系，也不被先后顺序所影响，因此不用考虑二维*二维矩阵的情况（任务之间不涉及先后，不涉及顺序关联，也就是没有非线性关联），但是有遗忘机制，是随着任务的时间逐渐减少权重，那么这就是线性递减，我们只需要把时间越久（或者任务排序越长的）
用次数作为权重衰减就好了。
具体步骤：
把每次任务产生的关键词序列[a,b,c,d,e,f,g.....]展开为二维（这一步是计算每个词之间每次的关联），用0-1矩阵M表示。那么M1代表第一次，M2代表第二次，我们把M1+M2，算出两次任务后，关键词累积的关联值。这种累积是根据次数统计，每次+1。算上时间系数，比如第n次累积值就是1-0.01*n，这样子相当于把三维矩阵压缩为二维矩阵。矩阵上的值就是累积的各个元素之间的关联值，也就是相关性。
ok了，以上就是不同关键词之间的相关性计算
下一步，计算推理关系
前面根据ai的回答，我们假设两个关键词分别出现的次数a，b： 分子（a）/分母(a+b)
 与 分子（b）/分母(a+b) ,表示各自在累积中占比，占比越多的那个关键词，越说明有他在，就大概率同时出现，
那就说明这个关键词对于它们同时在是充分条件。现实中我们知道，往往原因导致结果，虽然偶尔逆向，结果可能反向产生原因，但毕竟这种情况还是少数，因此，原因往往是结果的充分条件，a是b的充分条件，a就是原因，这就是推理计算的原理。
ok了，这下推理的思路也有了
接下来我们用py实现一个相关性矩阵（带推理的)存储库，输入一个关键词，计算出与其相关的关键词有哪些，并且与其是因还是果的关系（谁推导谁？）




算完的累积相关性矩阵后，输入一维向量关键词，通过与单个矩阵进行矩阵乘法计算，产生一次推理路径，多次反复计算，产生更深的推理路径
推理路径越深，逻辑性就越强（这些词的组合产生的逻辑性的概率就越大的意思）
每层推导，都可以计算出每层与每层之间的因果性，并进行记录，从而算出第n层，几个元素之间的推理路径



工程实现：
	0.首先，准备n组允许长度不一的向量序列，[a,b,c,...]a,b,c分别是长度不一的关键词向量
	1.把关键词序列展开成二维，并且因为是对称的原因，另一半就直接为0，减少后续计算，并且每次矩阵都要单独保留，每次累积计算都是重新计算，因为涉及到时间衰减
	2.多次产生关键词序列，也就多次累积二维矩阵（并且每次都是重新计算，因为考虑到了时间累减）
	3.足够多的次数了，假设我们的累积相关性矩阵M已经算好了，下一步我们算推理路径
	4.输入一组关键词向量n（展开为二维矩阵，因为算的是整体相关），计算M第一层推理路径
	5.关键词计算出M第一层n个元素的推理路径，并计算它们的因果路径X1（因果路径是在推理路径的基础上独立计算的,注意，无论路径多少层，都一直是计算M矩阵乘法，因为路径是同一时空的M,不会产生线性变换）
	6.同上关键词计算出M第二层少于n个元素的推理路径，以及它们的因果路径X2(此时有几个元素因为乘0，所以数量减少了，也就是掉队了，推理越深，还存留的推理路径数量就越少)

	7.反复计算，计算n次，推理路径的长度就是n，推理路径的质量理论上大部分就会越低（因为概率是越乘越小）。如果算到很深的层，整体概率相对还是很大，那么说明这个路径非常有价值，整体的关联度巨高（输入的关键词与计算出的推理路径，两者有相同的推理路径）
 这里要强调一下，如果是个体，相对推理路径衰减没那么快，组合衰减的比较快，但是如果组合相对比较高，那么说明这个组合的推理路径正是这个组合的强逻辑解释

以上的方法，我们叫他：无时空顺序的逻辑推理
